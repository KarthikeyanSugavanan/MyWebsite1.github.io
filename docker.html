Dockerized Encrypted Web Messaging Portal

Development of a Message Portal Web Application that can run within the docker container which encrypts and decrypts the message passed through the portal.


Subject : Virtualization

Tools Used : Docker , DockerHub , Python

Intalling and Setting Up Docker

The Docker Hub is like the GitHub of Docker Containers. To create our own private or public repository, we visit the Docker Hub website. we can use the Docker hub to quickly deploy our web applications. Using the Docker hub, we can push and pull changes of our web application across machines.

DOCKERFILE CREATION

Docker Containers are created from Docker Images. To get our web application distributed to different machines, we’ll need to create a Docker Image to send to them. This is done using a Docker file. A Docker file is usually around 3-30 lines long and contains all the commands needed to create our Docker Image.

Here's an example of a simple Docker file :
FROM docker/whalesay:latest
RUN apt-get -y update && apt-get install -y fortunes
CMD /usr/games/fortune -a | cowsay

DEPLOY APPLICATION ON DOCKER

Now we 've got our Docker machines running, Docker Hub repository set up, and Docker Image built, we are ready to deploy the web application across our machines.
First, we push our application to our repository.
Next, we'll pull our application from our repository to other machines. We do all of this with the docker run command. It will automatically find, pull, and run our new web application.
That's it! Those are the steps to use Docker to easily deploy an application to multiple machines.
