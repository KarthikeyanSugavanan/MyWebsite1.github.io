Online Food Ordering System

An application portal were customers can order food online from multiple restaurents . The system will allot work to the delivery boy / guy based on Circular Queue Data Structure which follows a FIFO principle.


Subject : Data Structure and Algorithms

Tools Used : Visual C++ and MySql

Welcome Page

The Application welcomes all users of the app with the welcome page also displays options such as offers , feedback and also a "Start" option to take to ordering page. All users can also login using the login button. Each button will take the user to its respective page , the "Start" button will require the user to again login in to his account if not already logged in before placing an order.

Login Page

This is the Login page , where all types of users such as the Admin , Customers and Delivery personnel . The Admin can create an account for himself/herself and can also create an account for deviverly personnel at the time of recruiting.

Sign-Up Page

Customers without an account can sign-up using clicking the SIGN-UP button provided in the Login Page.

Admin's Page

Once the user signs in as Admin throught the Login page . The Admin can view multiple options of viewing orders ,add/delete or update restaurant details and similarly recruit/dismiss or update delivery personnel's info.

In the oders tab , the admin can see all details of the order such as order-id , order number , customer-id , delivery personnel id , total amount of order , ordered food and location details.

Recruiting Page

Here, the admin can fill in details of the newly recruited delivery guy , the system will auto-generate an id for the new user and will update the data in the respective table/file.

Customer Ordering Page

Once the customer has logged into his account , he/she can use the drop down box to chooses various restarent , based on the restaurant select , the food drop down box will display the options availed at that restaurant.

The user then can increase or decrease the quntity of food or also cancel the item before moving to the payment page , using the interactive table present at right bottom of the screen.

Payment Portal

Here the user will can pay for the food either using the card option or by opting for COD (Cash on Delivery) option.

Once the Order is confirmed , the system will send the order to the delivery personnel , the system allots work to the dilivery personnel based on Circular Queue Data Structure Algorithm , the system also keeps a count on the number of trips made by each personnel , so no work is given a particular person repeatedly.

Delivery Boy Page

This is the Delivery boy page where the system will display orders to the person , once dilivered the person can intimate the system by logging into his/her page and updating the Delivery Status option.
